# ملف المشروع للرفع اليدوي

## الملف الجاهز

- **الاسم:** `sutra-project-full.zip`
- **المكان:** داخل مجلد المشروع نفسه  
  `c:\Users\DELL\OneDrive\Desktop\s\sutra-project-full.zip`

## ماذا يحتوي الملف؟

- كل الكود (مجلد `src`، الإعدادات، `package.json`، إلخ)
- مجلد **data** (المنتجات والأقسام الافتراضية)
- ملفات Netlify (`netlify.toml`) وملفات التوثيق
- **لا يحتوي:** `node_modules`، `.next`، `.git` (لتقليل الحجم)

## كيف تستخدمه للرفع اليدوي؟

### 1) إنشاء الملف من جديد (لو حذفته أو عدّلت المشروع)

من الطرفية داخل مجلد المشروع:

```powershell
cd c:\Users\DELL\OneDrive\Desktop\s
powershell -ExecutionPolicy Bypass -File .\zip-project.ps1
```

سيتم إنشاء أو استبدال `sutra-project-full.zip` في نفس المجلد.

### 2) رفعه على Netlify (ربط من GitHub وليس سحب ملف)

- Netlify **لا** يقبل رفع ملف zip مباشرة لنشر موقع Next.js كامل.
- الطريقة الصحيحة: ربط الريبو من **GitHub** (Import from Git) ثم Deploy.
- استخدم الملف **zip** لأي من التالي:
  - نسخ احتياطي للمشروع
  - إرسال المشروع لشخص آخر
  - فك الضغط في مكان آخر ثم ربط ذلك المجلد بـ Git ورفعه على GitHub، وبعدها ربط Netlify من GitHub

### 3) لو تريد نشر ثابت فقط (صفحة واحدة)

- المجلد **netlify-manual** فيه `index.html` جاهز.
- اسحب مجلد **netlify-manual** (أو محتوياته) إلى Netlify → Deploy manually ليعمل موقع ثابت بسيط.

---

**ملخص:** الملف `sutra-project-full.zip` هو نسخة كاملة من المشروع للنسخ الاحتياطي أو المشاركة. للنشر الكامل على Netlify استخدم ربط GitHub وليس رفع الـ zip.
